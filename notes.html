1. Web Basics

How the Web Works

The web is a set of computers that communicate with each other. To visit page, a computer sends an HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the computer where a web browser interprets the page and displays it on the screen. This video explains it.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks". Overview

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This video explains.

Why Computers are Literal

Some people call computers stupid because they interpret instructions literally. I don't mind.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

4. Python

Functions

.find takes one to three arguments: string to find, beginning location within string, ending location within string.

object oriented programming: the input to the left of ".find" is the object, and the inputs within the parenthetical are arguments or operands.

Debugging

informal: add print statements to functions, scripts, etc.

exponent operator: "**", not "^"

if an error thrown, check the last line of error message for the specific subroutine that failed. problem is here or upstream.

if error message is unhelpful, start with a working example snippet and alter it bit by bit until it crashes or succeeds

if no error is thrown, introduce print statements that output variables as the program runs

Miscellaneous

  • mutation and aliasing: mutation changes an element of a list or other mutable object (some objects are immutable), while aliasing does not change the original entity
  • methods vs functions: .append is a method. invoke it by using object-period-method syntax
  • + vs append: append is a method, + is an operator append appends a single element to the end of a list, while + operates on two lists, concatenating all individual elements into a new list
  • .index: LIST.index(VALUE) if VALUE is in LIST, .index returns the first position where VALUE is found in LIST, otherwise thows an error
  • in: VALUE in LIST: if VALUE is in LIST, returns 'True', otherwise 'False'
  • while loop best practices: highly recommended to first set up the structure and make sure that the count variable increases in each loop. avoids getting into infinite loops.
  • docstrings:
    def function(inputs)
    """this docstring will be appear as in some autocomplete dialogues"""
    return(output)
  • when inputing a list of arguments for a function, split the single list into a set of individual arguments by using * operator, calling function(*args) rather than function(arg1, arg2, etc) 
        def func1(x, y, z):
      print x
      print y 
      print z                 

    def func2(*args):
      # Convert args tuple to a list so we can modify it
      args = list(args)
      args[0] = 'Hello'
      args[1] = 'awesome'
      func1(*args)

    func2('Goodbye', 'cruel', 'world!')

    # Will print
    # > Hello
    # > awesome
    # > world!
  • "falling through" an if statements means moving to the bottom because all conditionals are false.
